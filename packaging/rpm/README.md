# RPM repository layout (US-015)

This directory is the source-of-truth for the `pkg.paneflow.dev/rpm/`
repository. It is not populated in git — the release workflow copies
built `.rpm` artifacts into `Packages/` and runs `createrepo_c` against
this directory tree.

## Layout

```
packaging/rpm/
├── Packages/                  # ← .rpm files land here at publish time
│   └── paneflow-<version>-1.x86_64.rpm
└── repodata/                  # ← generated by createrepo_c, not committed
    ├── repomd.xml
    ├── repomd.xml.asc         # ← GPG detached signature of the metadata (US-015)
    ├── primary.xml.gz
    ├── filelists.xml.gz
    └── other.xml.gz
```

## Client `.repo` config

A user adds this to `/etc/yum.repos.d/paneflow.repo` (the postinst script
in US-016 will automate this):

```ini
[paneflow]
name=PaneFlow
baseurl=https://pkg.paneflow.dev/rpm
enabled=1
gpgcheck=1
repo_gpgcheck=1
gpgkey=https://pkg.paneflow.dev/gpg
```

- `gpgcheck=1` — verifies each RPM's embedded signature (US-017 adds
  the signature via `rpmsign --addsign` in `release.yml`).
- `repo_gpgcheck=1` — verifies the `repomd.xml.asc` signature on the
  repository metadata before trusting any listed package (US-015).

## Pruning

Old RPM files accumulate in `Packages/` across publishes (R2 costs are
per-GB-stored, not per-file). A future story will add a retention policy
(e.g., keep the 5 most recent minor versions). For now, prune manually
with `rclone delete` if storage grows unbounded.
