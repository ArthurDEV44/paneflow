Tres rutas de instalación existen para Linux: un paquete `.deb`, un
AppImage y un tarball crudo. Elige el que coincida con tu
distribución y tu tolerancia a los paquetes gestionados por el
sistema. El AppImage es el predeterminado universal y no necesita
permisos de root.

<Callout type="tip">
El build de Linux necesita Vulkan. En Wayland, instala el cargador
de Vulkan de tu distribución (`libvulkan1` en la familia Debian,
`vulkan-loader` en Fedora, `vulkan-icd-loader` en Arch). El paquete
`.deb` declara esta dependencia y los demás requisitos de GUI por
ti; el AppImage y el tarball esperan que ya estén presentes.
</Callout>

## ¿Cómo instalo Paneflow en Ubuntu o Debian?

Descarga el `.deb` para tu arquitectura desde el [último release](https://github.com/ArthurDEV44/paneflow/releases/latest)
e instálalo con `apt`. El paquete declara sus dependencias de sistema
(`libvulkan1`, `libxcb1`, `libxkbcommon0`, `libfontconfig1` y algunas
más), así que `apt` trae todo lo que te falta.

```bash
curl -LO https://github.com/ArthurDEV44/paneflow/releases/download/v0.2.17/paneflow-0.2.17-x86_64.deb
sudo apt install ./paneflow-0.2.17-x86_64.deb
```

Para ARM64 (Raspberry Pi 5, Ampere, AWS Graviton), cambia `x86_64`
por `aarch64`:

```bash
curl -LO https://github.com/ArthurDEV44/paneflow/releases/download/v0.2.17/paneflow-0.2.17-aarch64.deb
sudo apt install ./paneflow-0.2.17-aarch64.deb
```

El binario aterriza en `/usr/bin/paneflow`. Desinstala más tarde con
`sudo apt remove paneflow`.

## ¿Cómo instalo Paneflow como AppImage?

El AppImage es un único ejecutable que corre en cualquier Linux
moderno. Sin root, sin gestor de paquetes, sin paso de instalación.

```bash
curl -LO https://github.com/ArthurDEV44/paneflow/releases/download/v0.2.17/paneflow-0.2.17-x86_64.AppImage
chmod +x paneflow-0.2.17-x86_64.AppImage
./paneflow-0.2.17-x86_64.AppImage
```

<Callout type="warn">
Ubuntu 24.04 y otras distribuciones recientes ya no incluyen FUSE 2
por defecto y rechazarán montar el AppImage. Si ves
`dlopen(): error loading libfuse.so.2`, ejecuta con el respaldo
extract-and-run:

```bash
./paneflow-0.2.17-x86_64.AppImage --appimage-extract-and-run
```

O instala FUSE 2 una sola vez con `sudo apt install libfuse2t64`.
</Callout>

Para lanzar desde cualquier sitio, mueve el AppImage a `~/.local/bin`
y renómbralo:

```bash
mkdir -p ~/.local/bin
mv paneflow-0.2.17-x86_64.AppImage ~/.local/bin/paneflow
chmod +x ~/.local/bin/paneflow
```

## ¿Cómo instalo Paneflow desde un tarball?

Usa el tarball cuando quieras control total sobre dónde vive el
binario o cuando estés guionando una instalación no interactiva.

```bash
curl -LO https://github.com/ArthurDEV44/paneflow/releases/download/v0.2.17/paneflow-0.2.17-x86_64.tar.gz
tar -xzf paneflow-0.2.17-x86_64.tar.gz
mkdir -p ~/.local/bin
mv paneflow ~/.local/bin/
chmod +x ~/.local/bin/paneflow
```

El tarball extrae un único binario `paneflow`. Déjalo en cualquier
sitio de tu `PATH`. Si `~/.local/bin` aún no está en tu `PATH`,
consulta la [sección de solución de problemas](#what-if-paneflow-is-not-found-in-my-path)
de abajo.

## ¿Cómo verifico la instalación?

Ejecuta `paneflow --version`. La salida esperada:

```text
paneflow 0.2.x
```

Si ves la versión impresa, el binario está en tu `PATH` y las
dependencias de GPU se resolvieron. Lanza el editor con:

```bash
paneflow
```

El primer arranque abre un espacio de trabajo vacío con un único
panel de shell. Presiona `Ctrl+Shift+D` para dividir horizontalmente
o `Ctrl+Shift+E` para dividir verticalmente.

## ¿Qué pasa si `paneflow` no se encuentra en mi PATH?

Si `paneflow --version` devuelve `command not found`, el binario no
está en el `PATH` de tu shell. Casi siempre esto significa que usaste
la ruta de AppImage o tarball y dejaste el binario en `~/.local/bin`,
que algunas distribuciones no ponen en el `PATH` por defecto.

Añádelo una vez por shell:

**bash** (`~/.bashrc`):

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

**zsh** (`~/.zshrc`):

```bash
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**fish** (`~/.config/fish/config.fish`):

```fish
fish_add_path ~/.local/bin
```

`fish_add_path` persiste en la variable universal `fish_user_paths`,
así que el cambio sobrevive a los reinicios. Abre un nuevo terminal
para que surta efecto en la sesión actual.

Luego vuelve a ejecutar `paneflow --version`. Si sigue fallando,
confirma que el binario es ejecutable (`ls -l ~/.local/bin/paneflow`
debe mostrar `-rwxr-xr-x`) y que copiaste el binario, no el
`.tar.gz`.
