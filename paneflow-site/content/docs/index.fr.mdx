Paneflow est un multiplexeur de terminal multiplateforme conçu pour
les workflows CLI agentiques - Claude Code, Codex, OpenCode et tout
autre agent qui parle shell natif. Volets divisés, sessions
persistantes, détection de ports de serveur de développement et
thématisation zéro-configuration, le tout dans une application native
accélérée par GPU qui démarre en moins de 200 ms.

<Callout type="info">
Paneflow distribue des binaires signés pour **macOS** (`.dmg` notarisé,
Apple Silicon) et **Linux** (`.deb`, AppImage, archive tar, x86_64 et
ARM64). Les boutons ci-dessus récupèrent le format recommandé pour
chaque OS ; les guides par OS ci-dessous couvrent tous les autres

## Ce qui tourne à l'intérieur

- **Rust pur** + [GPUI](https://www.gpui.rs/) - le même framework qui fait tourner Zed.
- **Émulation VT native** via `alacritty_terminal` (crates.io 0.26).
- **PTY natif** via `portable-pty` - même chemin de code sur Linux et macOS.
- **IPC JSON-RPC** sur `$XDG_RUNTIME_DIR/paneflow/paneflow.sock` pour le scripting sans interface.
- **Démarrage à froid sous 200 ms** et latence touche-au-pixel sous 4 ms sur la sonde de latence.

## Configuration requise

| Plateforme | Minimum | GPU |
